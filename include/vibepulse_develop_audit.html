3. 实时编码（线程/回调/延迟）↑ 显著提升
权重 30%预估 A4–A5
已修 main() 的 sleep_for(200ms) 轮询彻底消除。改为 pause()（阻塞等待信号），配合 SIGINT/SIGTERM 处理，主线程零 CPU 占用。
已修 ConsolePresenter 用 AppStateStore::waitForUpdate()+条件变量驱动打印，完全事件驱动，BPM 变化立刻响应，不再轮询。
已修 crossfade 的 sleep_for 改为 cv.wait_for，中断响应从 50ms 降至 <1ms，这正是之前讨论的修复方案。
新增 AudioService 类，BPM callback → AudioService::updateBpm() → ZoneMusicPlayer::updateBPM()，回调链路清晰无冗余锁。
monitorLoop 仍使用 sleep_for(200ms) 检测曲目是否播放完毕（轮询 isFinished）。这是一个合理的实现，但可在文档中注明：isFinished 属于状态轮询，因 SDL2_mixer 无曲目结束回调接口而采用此方式，200ms 对音乐切换场景可接受。否则老师可能认为这是对 sleep 的不当使用。建议文档说明
README 中延迟量化分析仍不完整。评分标准要求"quantitative assessment of latencies"——例如：sensor DRDY 中断→回调链→音乐切换的端到端延迟实测值。建议在 CODE_ARCHITECTURE.md 中补充一节具体数字。补文档
